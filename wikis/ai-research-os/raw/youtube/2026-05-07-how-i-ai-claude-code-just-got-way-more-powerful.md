# Claude Code Just Got WAY More Powerful

## Summary
Claire Vo summarizes Anthropic Code with Claude announcements around Claude Code routines, outcomes, managed multi-agent orchestration, Dreams memory, and expanded usage limits, framing them as practical steps toward Claude as an agent platform for builders.

The useful learning for Self-OS is that Anthropic is productizing several pieces of an agent operating system: scheduled/webhook-triggered routines, rubric-based outcomes, programmable multi-agent teams, and cross-session memory consolidation. These map directly onto recurring Hermes/Self-OS loops: daily briefs, newsletter generation, PRD review, wiki maintenance, agent handoffs, and quality gates.

## Key points
- **Routines turn Claude Code into an automation surface:** tasks can run on schedules, HTTP webhooks, or GitHub triggers, locally or remotely.
- **Outcomes formalize “done”:** managed agents can evaluate work against a rubric and iterate toward a target state, reportedly up to 20 iterations.
- **Multi-agent orchestration is becoming a first-class API primitive:** teams can be defined with roles, hierarchy, tools, shared workspace, and coordination patterns.
- **Dreams points toward durable agent memory:** Anthropic is experimenting with memory consolidation across sessions, similar to how Self-OS needs raw captures, compaction, and durable wiki/skill outputs.
- **The strategic direction is platformization:** Claude Code is no longer only an interactive coding assistant; it is moving toward a programmable agent runtime.

## Why it matters
This is relevant to the ai-research-os wiki because it captures a near-term product shift in coding agents: from chat-driven tools to scheduled, evaluated, multi-agent workflows. It also provides useful patterns for Hermes/Self-OS: routines map to cron jobs, outcomes map to rubrics and acceptance criteria, Dreams maps to memory compaction, and multi-agent orchestration maps to Kanban/subagent workflows.

## Self-OS / Hermes implications
- Treat **recurring routines** as a core operating-loop primitive, not an afterthought.
- Define more tasks with explicit **rubrics/outcomes** so autonomous agents can self-check before reporting completion.
- Use **multi-agent roles** when tasks split naturally into researcher, implementer, reviewer, and operator.
- Convert high-value captured learnings into durable **skills**, not just notes.
- Keep separating raw evidence, transient working state, and durable knowledge, because agent-memory products are converging on the same pattern.
