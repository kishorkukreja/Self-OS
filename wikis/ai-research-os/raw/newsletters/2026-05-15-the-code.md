## 💡 Insight — Why is agentic coding being called a trap

- **The orchestrator pitch is stalling.** Veteran developer Lars Faye's viral essay argues the plan-and-delegate workflow collapses because supervising AI requires the exact coding skills that heavy AI use erodes. [Essay](https://larsfaye.com/articles/agentic-coding-is-a-trap)
- **Anthropic's own data backs it.** A study showed a 47% drop in debugging ability among heavy AI users. Django co-creator Simon Willison admits he no longer has a clear mental map of apps he builds with agents. [Study](https://www.anthropic.com/research/how-ai-is-transforming-work-at-anthropic)
- **Not anti-AI.** Faye still uses LLMs for specs and ad-hoc tasks — his concern is specifically the "orchestration" model where AI owns implementation end-to-end.
- **The skill atrophy trap.** Outsourcing implementation leads to an inability to supervise AI effectively, since supervision requires the same skills being replaced. [Forbes](https://www.forbes.com/sites/juliakorn/2026/05/14/youre-getting-faster-and-dumber-how-to-fight-ai-skill-atrophy/)

## 🛠️ AI Coding Hack — How to speed up Claude's first API call

Send a throwaway `max_tokens=0` request before your real prompt to pre-warm the system prompt cache. According to Anthropic benchmarks, this makes the first real response **~52% faster** for a 160k-token system prompt. Apply `cache_control` to the system prompt block — not the placeholder message. Cache lasts 5 minutes; set a timer to retrigger if traffic is slow.

Source: [ClaudeDevs on X](https://x.com/ClaudeDevs/status/2055069548672631218)

```python
client.messages.create(
    model="claude-opus-4-7",
    max_tokens=0,
    system=[{
        "type": "text",
        "text": BIG_SYSTEM_PROMPT,
        "cache_control": {"type": "ephemeral"},
    }],
    messages=[{"role": "user", "content": "warmup"}],
)
```
