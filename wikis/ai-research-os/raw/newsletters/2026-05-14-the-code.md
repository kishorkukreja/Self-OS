## 💡 Insight — Why is every AI lab hiring forward-deployed engineers?

- Anthropic ($1.5B venture with Blackstone/Goldman Sachs), OpenAI (Deployment Company), and Google (hiring hundreds of FDEs) all announced on-site engineer programs within two weeks of each other.
- Palantir pioneered the Forward-Deployed Engineer role a decade ago — FDEs ship actual production code at client sites, unlike consultants who deliver presentations.
- **Knowledge gap bet:** Customer engineers know their data and compliance needs; vendor engineers know the AI architecture. FDEs bridge this by writing code directly in the customer's environment.
- **Stickiness bet:** Once an FDE team spends six months embedding AI into a firm's workflow, it becomes critical infrastructure nearly impossible to remove, securing steady recurring revenue.

## 🛠️ AI Coding Hack — How to ship code from Slack using Cursor

Once an admin connects Cursor via **Dashboard → Integrations**, mention `@Cursor` in any Slack channel to trigger an agent task. The agent reads the thread, runs in a sandbox, opens a PR, and pings you when done — ideal for clearing small backlog items without opening the editor.

```
@Cursor write a migration to add a discount_code column to the orders table, follow the pattern in db/migrations
```

Source: [Swizec's blog](https://swizec.com/blog/cursor-background-agents-in-slack-changed-my-workflow/) | More hacks: [hackbook](https://hackbook-chi.vercel.app/)
