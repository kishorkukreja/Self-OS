## 💡 Insight — The rise of the personal AI agent that never forgets

- **Sessions are dying.** Agents like OpenClaw and Hermes Agent run 24/7 as background services with months of persistent memory — unlike Claude Code or Codex, which wipe context each session.
- **Two species are emerging.** OpenClaw (345K GitHub stars) bets on breadth with wide integrations; [Hermes Agent](https://nousresearch.com) from Nous Research bets on depth with persistent memory and a closed learning loop.
- **Security is shaky.** OpenClaw was hit by a coordinated attack (341 malicious registry entries); Microsoft warned enterprise customers to avoid it on work machines.
- **Memory is the moat.** Whoever controls the memory leads the next wave of developer tools. See what's [already being built](https://hermes-agent.nousresearch.com/docs/user-stories) with Hermes.

## 🛠️ AI Coding Hack — How to stop losing context between Claude Code sessions

Every Claude Code session times out and the next one starts from scratch. [Matt Pocock](https://x.com/mattpocockuk/status/2052489881088049407) (ex-Vercel) created a `/handoff` skill that compresses your session into a Markdown file so the next agent starts with full context.

**Install:**
```
npx skills@latest add mattpocock/skills
```
Select "handoff", set Claude Code as your agent, and restart.

**Usage** — before closing a session:
```
/handoff debug the failing auth tests
```
When you start your next session, load that Markdown file and Claude will resume right where you left off.

Source: [x.com](https://x.com/mattpocockuk/status/2052489881088049407) | [Skill file](https://github.com/mattpocock/skills/blob/733d312884b3878a9a9cff693c5886943753a741/skills/in-progress/handoff/SKILL.md)
