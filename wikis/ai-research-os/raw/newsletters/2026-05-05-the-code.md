## 💡 Insight — Why is GitHub breaking when its competitors aren't?

**X marks the exit.** HashiCorp founder Mitchell Hashimoto logged daily GitHub outages for a month before [walking away](https://mitchellh.com/writing/ghostty-leaving-github) after 18 years, calling the platform "no longer a place for serious work." GitHub's CTO has [admitted](https://github.blog/news-insights/company-news/an-update-on-github-availability/) the platform wasn't built for AI agent load.

- Vercel, Sentry, and Railway handle the same agent traffic daily without issue — the volume has grown steadily for two years, not overnight.
- Google was already preparing for a 10x code volume spike by mid-2025; GitHub didn't start its own plan until October, and by February the target had tripled.
- Two decades of tech debt and a [half-finished Azure migration](https://newsletter.pragmaticengineer.com/p/the-pulse-github-breaks) are slowing every fix.
- The default has broken. Most users won't switch due to ecosystem lock-in, but [GitLab](https://about.gitlab.com/) is back in the conversation — GitHub must earn trust from the ground up for the agent era.

## 🛠️ AI Coding Hack — How to secure your Claude Code projects

Claude runs shell commands (like `npm install`) with your full user permissions by default. A malicious package could read SSH keys, copy `.env` files, or access anything in your home directory.

Run this command inside Claude Code:

```
/sandbox
```

Select **"accept-edits mode."** Claude can now read, write, and run commands within your project folder, but the OS immediately blocks any attempt to touch files outside that scope.
