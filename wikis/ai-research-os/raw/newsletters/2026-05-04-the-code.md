## 💡 Insight — Cursor's bet: the agent harness is the product

**Beyond the model.** Cursor released an [SDK](https://cursor.com/blog/typescript-sdk) that positions their agent harness — the tools, prompts, and edit logic surrounding the model — as the core product, not the model itself.

- OpenAI and Claude models use different file-editing formats; Cursor [tailors output](https://cursor.com/blog/continually-improving-agent-harness) to each model to save reasoning tokens and reduce errors.
- Custom metric "Keep Rate" measures how much AI-generated code survives user edits; every update is A/B tested.
- One sprint cut unexpected tool call errors by 10x; targeted prompting fixed Claude's avoidance of long-context tasks.
- Nicolas Bustamante showed that Claude Code, Codex CLI, and Copilot CLI produce [totally different results](https://x.com/nicbstme/status/2051131906327212298) from the same underlying model.
- Each new model breaks the previous harness — maintaining these layers is ongoing, never-ending work.

## 🛠️ AI Coding Hack — How to skip copy-pasting errors into Claude Code

Instead of manually copy-pasting error logs (losing formatting and risking truncation), pipe any command's output directly into Claude:

```bash
cat error.log | claude "fix this"
```

The model gets the full `stdout` exactly as it appears in your terminal — much better context than a messy paste. Works for any command:

```bash
npm test 2>&1 | claude "diagnose the failures"
```

Speed it up with a shell alias:

```bash
alias fix='claude "fix this"'
```
