## 💡 Insight — Devs love SKILL.md files — but there's a hidden cost

- Most engineers write one massive SKILL.md that consumes ~20% of context window before the agent starts.
- Restructuring as a lean "spine" pointing to reference files drops token cost to ~7% — same content, better architecture.
- Anthropic's runtime loads SKILL.md in stages: metadata on every turn, main body only when a skill triggers, references only when specifically called.
- Skills tuned for one model often fail after upgrading — "smarter" models follow instructions too literally and stop exercising judgment.
- Every SKILL.md is effectively a small program: re-run evals after every model switch.

## 🛠️ AI Coding Hack — How to prevent leaking secrets in Claude Code

A 3-line pre-commit hook pipes staged diffs to Claude for review before every commit. Save this to `.git/hooks/pre-commit` and run `chmod +x .git/hooks/pre-commit`:

```bash
#!/bin/bash
git diff --cached | claude -p "Spot any console.logs, hardcoded secrets, or unfinished TODOs in this diff?"
```

Every `git commit` sends staged changes to Claude. The review appears before the commit completes — hit `Ctrl+C` if Claude flags a real issue.
