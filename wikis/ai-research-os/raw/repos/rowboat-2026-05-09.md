# Rowboat — open-source AI coworker with memory

## Summary
Rowboat is an open-source, local-first AI coworker that connects to work data such as email, calendar, Drive, meeting notes, and voice memos, then turns that context into an editable knowledge graph. Its key architectural choice is to store working memory as an Obsidian-compatible Markdown vault with backlinks, rather than hiding memory inside a hosted model or opaque vector store. It can use this memory to produce artifacts such as meeting briefs, emails, docs, plans, summaries, and PDF decks.

For Self-OS, Rowboat is highly relevant because it overlaps with several active design themes: durable local memory, Markdown as the user-owned system of record, agent workflows that act on accumulated context, and AI operating-system patterns for personal/professional work. The useful learning is not merely “another AI assistant,” but the productization of **context accumulation → knowledge graph → action** as a local-first coworker.

## Key points
- **Local-first working memory:** Rowboat stores user context locally as plain Markdown, making memory inspectable, editable, portable, and deletable.
- **Knowledge graph over cold retrieval:** Instead of reconstructing context from scratch for each prompt, it accumulates entities, relationships, decisions, commitments, and topics over time.
- **Obsidian-compatible vault:** The memory layer uses Markdown files and backlinks, aligning with personal-knowledge-base workflows and Self-OS wiki conventions.
- **Action-oriented coworker:** It is designed to generate practical outputs — decks, briefs, emails, docs, plans, voice notes — from accumulated context.
- **Work data connectors:** It supports Google services such as Gmail, Calendar, and Drive, plus optional Deepgram, ElevenLabs, Exa search, MCP servers, and Composio tools via local config.
- **AI operating-system signal:** Rowboat is evidence that the “personal AI OS” category is converging around local memory, work-context ingestion, graph-backed reasoning, and controlled action execution.

## Why it matters for ai-research-os
Rowboat is a concrete reference implementation for **local-first agent memory** and **AI coworker UX**. It provides a useful comparison point for Self-OS: both systems emphasize durable Markdown, inspectable state, and compounding knowledge. Rowboat appears more productized around desktop coworker workflows and SaaS connectors, while Self-OS currently prioritizes Git-backed wikis, cron loops, Telegram control flow, and agent/Kanban operations.

## What Rowboat Is

Rowboat is a **local-first AI coworker** designed to remember work context over time and act on it.

It connects to sources such as email and meeting notes, turns that information into a **long-lived knowledge graph**, and uses the graph to help with writing, planning, preparation, summarization, and artifact generation.

The memory layer is an **Obsidian-compatible vault** made of plain Markdown notes with backlinks, meaning users can inspect, edit, back up, and delete their data directly.

---

## Core Capabilities

Rowboat can:

- **Remember** important context:
  - People
  - Projects
  - Decisions
  - Commitments
- **Understand** what is relevant in the moment:
  - Before meetings
  - While replying to emails
  - While writing documents
- **Help users act** by producing:
  - Briefs
  - Emails
  - Documents
  - PDF slides
  - Summaries
  - Plans
  - Follow-up lists

---

## Example Use Cases

Rowboat can be used for:

- **Meeting preparation**
  - Pulls prior decisions, discussion threads, open questions, and relevant context.
- **Email drafting**
  - Drafts replies grounded in past communication and commitments.
- **Docs and decks**
  - Generates documents and PDF slide decks using existing context.
- **Follow-ups**
  - Captures decisions, action items, and owners.
- **Local workflows**
  - Creates files, summarizes into notes, and runs workflows using local tools.
- **Voice memos**
  - Records voice notes and updates key takeaways in the knowledge graph.
- **Knowledge graph editing**
  - Lets users visualize, edit, and update their graph because it is Markdown-based.

---

## Live Notes

**Live notes** are Markdown notes that update automatically.

Users can create one by typing:

```text
@rowboat
```

Live notes can:

- Track a competitor or market topic across X, Reddit, and news
- Monitor a person, project, or deal across the web or user communications
- Maintain a running summary of any subject

Important behavior:

- Everything is written back into the local Markdown vault.
- Users control what runs and when.

## Setup and Optional API Keys

### Google Setup

To connect Google services such as **Gmail**, **Calendar**, and **Drive**, follow:

- [`google-setup.md`](https://github.com/rowboatlabs/rowboat/blob/main/google-setup.md)

Google integration uses an OAuth authorization code flow requiring:

- Client ID
- Client Secret
- Localhost redirect URI

### Voice Input

To enable voice input and voice notes, add a **Deepgram** API key:

```text
~/.rowboat/config/deepgram.json
```

### Voice Output

To enable voice output, add an **ElevenLabs** API key:

```text
~/.rowboat/config/elevenlabs.json
```

### Web Search

To enable Exa research search, add an **Exa** API key:

```text
~/.rowboat/config/exa-search.json
```

### External Tools

To enable external tools, add either:

- Any MCP server
- Composio tools via API key

Composio API key location:

```text
~/.rowboat/config/composio.json
```

All API key files use this format:

```json
{
  "apiKey": "<key>"
}
```

## What it does

Rowboat is a **local-first AI coworker** that can:
- **Remember** the important context you don’t want to re-explain (people, projects, decisions, commitments)
- **Understand** what’s relevant right now (before a meeting, while replying to an email, when writing a doc)
- **Help you act** by drafting, summarizing, planning, and producing real artifacts (briefs, emails, docs, PDF slides)

Under the hood, Rowboat maintains an **Obsidian-compatible vault** of plain Markdown notes with backlinks — a transparent “working memory” you can inspect and edit.

## Integrations

Rowboat builds memory from the work you already do, including:
- **Gmail** (email)
- **Google Calendar** 
- **Rowboat meeting notes** or **Fireflies**

It also contains a library of product integrations through Composio.dev

## How it’s different

Most AI tools reconstruct context on demand by searching transcripts or documents.

Rowboat maintains **long-lived knowledge** instead:
- context accumulates over time
- relationships are explicit and inspectable
- notes are editable by you, not hidden inside a model
- everything lives on your machine as plain Markdown

The result is memory that compounds, rather than retrieval that starts cold every time.

## What you can do with it

- **Meeting prep** from prior decisions, threads, and open questions
- **Email drafting** grounded in history and commitments
- **Docs & decks** generated from your ongoing context (including PDF slides)
- **Follow-ups**: capture decisions, action items, and owners so nothing gets dropped
- **On-your-machine help**: create files, summarize into notes, and run workflows using local tools (with explicit, reviewable actions)

## Live notes

Live notes are notes that stay updated automatically. You can create one by typing '@rowboat' on a note. 

- Track a competitor or market topic across X, Reddit, and the news
- Monitor a person, project, or deal across web or your communications
- Keep a running summary of any subject you care about

Everything is written back into your local Markdown vault. You control what runs and when.

## Bring your own model

Rowboat works with the model setup you prefer:
- **Local models** via Ollama or LM Studio
- **Hosted models** (bring your own API key/provider)
- Swap models anytime — your data stays in your local Markdown vault

## Extend Rowboat with tools (MCP)

Rowboat can connect to external tools and services via **Model Context Protocol (MCP)**.
That means you can plug in (for example) search, databases, CRMs, support tools, and automations - or your own internal tools.

Examples: Exa (web search), Twitter/X, ElevenLabs (voice), Slack, Linear/Jira, GitHub, and more.

## Local-first by design

- All data is stored locally as plain Markdown
- No proprietary formats or hosted lock-in
- You can inspect, edit, back up, or delete everything at any time
