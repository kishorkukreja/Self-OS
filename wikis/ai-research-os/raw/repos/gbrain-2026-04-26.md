## Overview

GBrain is Garry Tan's opinionated, open-source agent brain. Built on Bun, it uses a **pluggable dual-engine architecture** (embedded PGLite or Postgres/Supabase) to store markdown and code pages as compiled truth + append-only timeline, with automatic typed-link extraction (`works_at`, `advises`, `founded`, `invested_in`, `attended`, `mentions`).

---

## Search & Retrieval (v0.22.0)

**Hybrid pipeline** fuses:
- pgvector HNSW cosine similarity
- chunk-grain tsvector FTS
- Reciprocal Rank Fusion

**Source-aware ranking:** SQL CASE expressions boost curated sources and dampen noise:
- `originals/` → 1.5× boost
- `writing/` → 1.4× boost
- `people/` → 1.2× boost
- `wintermute/chat/` → 0.5× dampen
- `daily/` → 0.8× dampen
- Hard-excludes: `test/`, `archive/`

`detail=high` gate bypasses boosts for temporal queries.

**Result:** +3.3 pts top-1 on Cat 13b benchmark, 39 unit tests, engine-parity coverage between Postgres/PGLite.

## Code Intelligence: Cathedral I & II

### Cathedral I (v0.19.0)
- Tree-sitter chunking for 29 languages via 36 embedded WASM grammars
- Incremental chunking (~95% embedding cost reduction)
- Doc↔impl linking
- `code-def` / `code-refs` CLI surfaces
- Schema: `page_kind` + chunk-level metadata (`language`, `symbol_name`, `symbol_type`)

### Cathedral II (v0.20.0)
- Call-graph edge capture: `code_edges_chunk` (resolved) + `code_edges_symbol` (unresolved)
- Parent-scope nested-chunk emission (methods as separate chunks with `parentSymbolPath`)
- Chunk-grain FTS with `search_vector`
- Two-pass structural retrieval (`--walk-depth 1-2`, `--near-symbol`) expanding results via callers/callees with `1/(1+hop)` decay
- New commands: `code-callers`, `code-callees`, `reconcile-links`, `reindex-code`
- `CHUNKER_VERSION` gates force full re-sync on format changes

## Key Concepts

- **Compiled truth + append-only timeline** — knowledge base structure
- **Typed-link extraction** — automatic relationship inference
- **SkillPacks** — portable bundles of skills, triggers, scripts, tests installable into any agent setup
- **`gbrain doctor --fix`** — auto-repairs DRY violations, replaces duplicated blocks with convention references
- **10-step skillify checklist** — built into gbrain doctor for skill quality gates
