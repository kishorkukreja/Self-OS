    17|## Overview
    18|
    19|GBrain is Garry Tan's opinionated, open-source agent brain. Built on Bun, it uses a **pluggable dual-engine architecture** (embedded PGLite or Postgres/Supabase) to store markdown and code pages as compiled truth + append-only timeline, with automatic typed-link extraction (`works_at`, `advises`, `founded`, `invested_in`, `attended`, `mentions`).

    23|## Search & Retrieval (v0.22.0)
    24|
    25|**Hybrid pipeline** fuses:
    26|- pgvector HNSW cosine similarity
    27|- chunk-grain tsvector FTS
    28|- Reciprocal Rank Fusion
    29|
    30|**Source-aware ranking:** SQL CASE expressions boost curated sources and dampen noise:
    31|- `originals/` → 1.5× boost
    32|- `writing/` → 1.4× boost
    33|- `people/` → 1.2× boost
    34|- `wintermute/chat/` → 0.5× dampen
    35|- `daily/` → 0.8× dampen
    36|- Hard-excludes: `test/`, `archive/`
    37|
    38|`detail=high` gate bypasses boosts for temporal queries.
    39|
    40|**Result:** +3.3 pts top-1 on Cat 13b benchmark, 39 unit tests, engine-parity coverage between Postgres/PGLite.

    44|## Code Intelligence: Cathedral I & II
    45|
    46|### Cathedral I (v0.19.0)
    47|- Tree-sitter chunking for 29 languages via 36 embedded WASM grammars
    48|- Incremental chunking (~95% embedding cost reduction)
    49|- Doc↔impl linking
    50|- `code-def` / `code-refs` CLI surfaces
    51|- Schema: `page_kind` + chunk-level metadata (`language`, `symbol_name`, `symbol_type`)
    52|
    53|### Cathedral II (v0.20.0)
    54|- Call-graph edge capture: `code_edges_chunk` (resolved) + `code_edges_symbol` (unresolved)
    55|- Parent-scope nested-chunk emission (methods as separate chunks with `parentSymbolPath`)
    56|- Chunk-grain FTS with `search_vector`
    57|- Two-pass structural retrieval (`--walk-depth 1-2`, `--near-symbol`) expanding results via callers/callees with `1/(1+hop)` decay
    58|- New commands: `code-callers`, `code-callees`, `reconcile-links`, `reindex-code`
    59|- `CHUNKER_VERSION` gates force full re-sync on format changes
