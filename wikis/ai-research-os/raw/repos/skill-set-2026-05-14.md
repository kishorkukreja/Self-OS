## Skill details captured

### create-onboarding-video

Purpose: Produce a short, punchy iOS onboarding video in Remotion that showcases one feature working. The output should feel like an App Store preview zoomed into the moment that proves the feature works — not a tutorial, not a screen recording, not a generic marketing reel.

Core constraints:

- 3–8 seconds per onboarding screen; whole video rarely exceeds ~30 seconds.
- Never render the whole UI as the main focus; isolate relevant components.
- One beat should demonstrate one thing the feature does.
- Use cropped screenshots, masks, tinted backgrounds, transitions, springs, and shared-element motion.
- No voiceover by default; short captions are acceptable.
- Captions should be consistent, readable, top-positioned, and visible for the whole beat.
- Interactive beats require visible cursor movement leading the tap/click/selection.
- Stills are required; do not invent UI from text alone.
- One feature per video; split unrelated features into separate videos.

```markdown
# Create Onboarding Video

Produce a short, punchy iOS onboarding video in Remotion that showcases one feature working. Output is meant to feel like an App Store preview zoomed into the moment that proves the feature works — not a tutorial, not a screen recording, not a marketing reel.

## What you make

- Length: short. 3–8 seconds per onboarding screen, stitched together. Whole video rarely exceeds ~30s.
- Style: UI-first, never the whole screen. Each beat shows a piece of the feature in action — a single button being tapped, a toggle flipping, a row reordering, a sheet sliding up, a chart filling in — animated with nice UI-like transitions.
- What pieces means: crop, mask, or extract just the relevant component from the supplied still — the card, the input field, the tab bar, the empty state turning into a filled state. The rest of the UI is omitted, blurred, or implied by a tinted background.
- Tone: to the point. Each beat communicates one thing the feature does.
- Output: a Remotion project that renders to MP4.

## Workflow

1. Intake — ask for stills + intent.
2. Plan the shots.
3. Build with Remotion.
4. Iterate.

## Operating rules

- Stills are required.
- Pieces of the UI, not the whole UI.
- One feature per video.
- Show, don't narrate.
- Captions are visible for the entire beat.
- Cursor leads every tap.
- Match the app's design language.
- Ask when in doubt.
```
