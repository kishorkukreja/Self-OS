# Obscura

## Summary

Obscura is an open-source, Rust-based headless browser engine designed for AI agents, web scraping, and large-scale browser automation. It runs real JavaScript through V8, exposes the Chrome DevTools Protocol, and is intended to work as a drop-in replacement for headless Chrome with tools such as Puppeteer and Playwright. Its core pitch is a small single binary with lower memory usage, faster startup, and optional stealth/anti-detection capabilities.

For SelfOS/Hermes, this is potentially useful as a lighter browser automation runtime for extraction, scraping, QA loops, and agent browsing tasks where launching full Chrome is expensive or operationally awkward. It should be treated as an experimental candidate until verified against difficult sites, CDP compatibility edge cases, authenticated flows, and dynamic apps.

## Key points

- Built as a lightweight headless browser engine for automation rather than desktop browsing.
- Runs JavaScript using **V8**.
- Supports the **Chrome DevTools Protocol**.
- Designed to connect from **Puppeteer** and **Playwright** via CDP.
- Ships as a single binary with no Chrome, Node.js, or runtime dependency requirement for basic CLI usage.
- Includes optional stealth mode for anti-detection and tracker blocking.
- CLI supports page fetches, DOM evaluation, HTML/link dumps, wait conditions, CDP server mode, and parallel scraping.

## Why it matters

Obscura fits a growing pattern in AI tooling: agents need browser access, but full Chrome can be heavy, slow, and operationally brittle at scale. A small CDP-compatible browser runtime could be valuable for:

- **Agent browsing:** giving agents a cheaper browser runtime for reading dynamic pages.
- **Knowledge ingest:** extracting pages where static HTTP extraction fails, without always launching full Chrome.
- **Web QA:** running simple browser checks through Playwright/Puppeteer-compatible APIs.
- **Research automation:** parallel scraping with lower memory overhead.
- **Sandboxed execution:** pairing with Sandcastle-style isolated agent runs where a single binary is easier than maintaining Chrome installs.

## Usefulness to Hermes / SelfOS

Yes — this could be useful, especially as an **optional browser backend** rather than a replacement for existing Hermes browser tools.

Best-fit use cases:

1. **Fallback extractor for difficult pages**  
   If `web_extract` returns incomplete content and built-in browser tooling is too heavyweight or unavailable, Obscura could be tested as a CLI/CDP-based fallback.

2. **Cheaper browser automation inside sandboxed agents**  
   A single binary is easier to drop into isolated Sandcastle-style runs than a full Chrome/Chromium stack.

3. **Parallel scraping jobs**  
   Its `scrape` command with concurrency could be useful for batch collection jobs, provided rate limits and robots/legal constraints are respected.

4. **Playwright/Puppeteer compatibility layer**  
   Existing Playwright/Puppeteer scripts may be able to connect over CDP without major rewrites.

5. **Agent QA loops**  
   For lightweight page-load and DOM checks, it could reduce the overhead of spinning up Chrome.

Cautions:

- The benchmark claims should be independently verified in the local environment.
- CDP support may not cover every edge case that Chrome supports.
- “Stealth” capabilities should be used conservatively and ethically.
- For visual QA, screenshots, layout debugging, and full browser fidelity, real Chrome/Chromium may still be better.
- It is currently best categorized as promising infrastructure, not a default dependency.

## Installation notes

Download prebuilt Linux binary:

```bash
curl -LO https://github.com/h4ckf0r0day/obscura/releases/latest/download/obscura-x86_64-linux.tar.gz
tar xzf obscura-x86_64-linux.tar.gz
./obscura fetch https://example.com --eval "document.title"
```

Build from source:

```bash
git clone https://github.com/h4ckf0r0day/obscura.git
cd obscura
cargo build --release

# With stealth mode
cargo build --release --features stealth
```

Requirements:

- Rust 1.75+
- First build can take several minutes because V8 compiles from source.

## Example usage

Fetch a page title:

```bash
obscura fetch https://example.com --eval "document.title"
```

Extract links:

```bash
obscura fetch https://example.com --dump links
```

Render JavaScript and dump HTML:

```bash
obscura fetch https://news.ycombinator.com --dump html
```

Wait for dynamic content:

```bash
obscura fetch https://example.com --wait-until networkidle0
```

Start CDP server:

```bash
obscura serve --port 9222
```

Start CDP server with stealth mode:

```bash
obscura serve --port 9222 --stealth
```

Parallel scrape:

```bash
obscura scrape url1 url2 url3 \
  --concurrency 25 \
  --eval "document.querySelector('h1').textContent" \
  --format json
```

## Playwright connection example

```javascript
import { chromium } from 'playwright-core';

const browser = await chromium.connectOverCDP({
  endpointURL: 'ws://127.0.0.1:9222',
});

const page = await browser.newContext().then(ctx => ctx.newPage());
await page.goto('https://en.wikipedia.org/wiki/Web_scraping');
console.log(await page.title());

await browser.close();
```

## Raw content

Obscura describes itself as “the open-source headless browser for AI agents and web scraping.” It is a lightweight, stealth-focused headless browser engine written in Rust. It is designed for web scraping and AI agent automation, runs real JavaScript via V8, supports the Chrome DevTools Protocol, and acts as a drop-in replacement for headless Chrome with Puppeteer and Playwright.

The README positions Obscura as optimized for automation at scale rather than desktop browsing. It compares itself against headless Chrome with claimed lower memory usage, smaller binary size, built-in anti-detection, faster page loads, faster startup, and compatibility with Puppeteer and Playwright.

Installation is available through GitHub Releases as prebuilt binaries for Linux, macOS Apple Silicon, macOS Intel, and Windows. It can also be built from source with Rust 1.75+. Optional stealth mode can be enabled with the `stealth` feature.

The CLI supports page fetching, JavaScript evaluation, link dumps, HTML dumps, dynamic-content waits, a CDP server, and parallel scraping. It includes examples for connecting Puppeteer and Playwright through CDP, as well as handling form submission, POST requests, redirects, and cookie persistence.
