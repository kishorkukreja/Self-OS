# UI Skills Directory for Design Engineers

## Summary
UI Skills is a directory of design-engineering skills for frontend teams and AI coding workflows. The directory describes itself as a thoughtful collection of 109 high-quality skills covering accessibility, motion, frameworks, testing, performance guidance, systems, visual design, interaction, craft, and taste.

The resource is relevant because it packages frontend and design-engineering know-how as reusable skills, mirroring the broader shift from ad hoc prompting toward skill-based agent workflows. The catalog includes skills for design exploration, production-grade frontend design, accessibility remediation, WCAG audits, metadata fixes, web-quality audits, motion performance, and UI review.

## Key points
- UI Skills catalogs 109 skills aimed at design engineers and frontend teams.
- Topics include accessibility, motion, systems, visual design, interaction, performance, craft, taste, frameworks, testing, and quality guidance.
- Example skills include `design-lab`, `canvas-design`, `frontend-design`, `bencium-innovative-ux-designer`, `interface-design`, `emil-design-eng`, `ui-ux-pro-max`, and `rams`.
- Accessibility skills include AccessLint workflows, `fixing-accessibility`, and `wcag-audit-patterns`.
- Web quality skills include `web-quality-audit`, `web-design-guidelines`, `fixing-metadata`, and other audit/remediation workflows.
- The directory is connected to the `ibelick/ui-skills` GitHub repository.

## Why it matters
This is useful for Self-OS and coding-agent workflows because it gives agents reusable frontend/design guidance rather than relying on generic “make it look good” prompts. It is especially relevant to UI generation, design QA, accessibility, and frontend polish.

## Raw content
The UI Skills directory page headline says: “Discover today's top UI skills.” It describes the site as a thoughtful directory of 109 high-quality skills for design engineers and frontend teams, including accessibility, motion, frameworks, testing, and performance guidance.

Primary topic filters include Accessibility, Motion, Systems, Visual, Interaction, Performance, Craft, Taste, and More. The catalog includes design exploration and product/UI design skills such as `design-lab` by 0xdesign, described as an interactive design exploration workflow using interviews, variants, and iterative UI refinement through feedback. Anthropic entries include `canvas-design` for creating original visual designs and art on digital canvases using form, space, and color, and `frontend-design` for building distinctive, production-grade frontend interfaces with polished code and non-generic visual quality.

Other product/UI design skills include `bencium-innovative-ux-designer` for production-grade frontend UI generation, `interface-design` for dashboards, admin panels, and SaaS apps, `emil-design-eng` for Emil Kowalski's design-engineering philosophy, `ui-ux-pro-max` with broad UI/UX design intelligence, and `rams` for real-time design feedback on accessibility, spacing, typography, contrast, and component quality.

Accessibility and compliance entries include `audit-and-fix`, `contrast-checker`, `link-purpose`, `refactor`, `use-of-color`, `fixing-accessibility`, and `wcag-audit-patterns`. These skills cover accessibility auditing, remediation, ARIA labels, keyboard navigation, focus management, contrast, form errors, and WCAG 2.2 workflows.

Web quality and metadata skills include `web-quality-audit` by Addy Osmani, `web-design-guidelines` entries, and `fixing-metadata`, which covers titles, meta descriptions, canonical URLs, Open Graph, Twitter cards, favicons, JSON-LD, and robots directives.
