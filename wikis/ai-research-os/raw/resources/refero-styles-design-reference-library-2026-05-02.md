# Refero Styles — Design Reference Library for AI Agents

## Summary

Refero Styles is a curated design-reference search tool for extracting and browsing design taste from real brands and products. It lets users search references by brand, mood, color, typography, or URL, then inspect styles in terms of colors, type, spacing, components, and a `DESIGN.md` file that an AI agent can use. For Self-OS design creation workflows, this is useful as a practical inspiration and prompt/context source when asking coding agents to produce higher-taste UI rather than generic AI-looking interfaces.

## Key points

- **Core positioning:** “Design taste, extracted.”
- **Status:** Beta.
- **Primary use case:** Search curated real-product references and convert taste into agent-usable design context.
- **Search axes:** brand, mood, color, typography, and URL.
- **Suggested categories:** Minimal Design, Clean SaaS, Editorial Type, Soft Gradients, Monochrome UI, Playful Design, High Contrast, Premium Design.
- **Sorting/filtering:** Trending, Popular, Newest.
- **Each style appears to expose:** colors, typography, spacing, components, and a `DESIGN.md` for agents.
- **Agent relevance:** complements the `design-md` skill by providing real-world style references and DESIGN.md examples that can be passed into coding/design agents.
- **MCP relevance:** promotes Refero MCP, which gives AI coding agents searchable access to product screens and user flows.

## Why it matters for Self-OS

Refero Styles should be treated as a design-creation resource rather than a generic article. It is especially relevant when building:

- landing pages
- product UI prototypes
- SaaS dashboards
- agent control panels
- DESIGN.md style specs
- prompt packs for design agents
- Claude/Codex/Cursor UI generation workflows

This can help reduce generic AI UI output by grounding agents in real, curated product aesthetics. The strongest Self-OS use is to pair it with the `design-md`, `claude-design`, `impeccable`, and `popular-web-designs` skills: select a Refero style, extract its DESIGN.md/style tokens, then use those constraints as the visual identity for generated interfaces.
