# Persistent Agent Pools

**Definition:** Persistent agent pools are groups of stateful subagents that can receive follow-up messages and retain task context across steps.

**Why it matters:** They enable richer coordination than one-shot delegation but require explicit status tracking, cancellation, and routing discipline.
