# Spec-Driven Development

**Definition:** Spec-driven development makes specifications, principles, plans, and task lists executable inputs for coding agents.

**Why it matters:** This concept appears in the newest compile as a practical operating pattern rather than a purely abstract term. It is useful for Self-OS because it describes how agent systems, research workflows, or operating cadences can be made more reliable when the system names the mechanism, captures evidence, and connects the source back to repeatable practices.
