# Software Fundamentals Matter More Than Ever

**Type:** video
**Date:** 2026-04-26
**URL:** https://www.youtube.com/watch?v=v4F1gFy-hqg
**Speaker:** Matt Pocock
**Raw file:** [[../../raw/youtube/2026-04-26-matt-pocock-software-fundamentals-matter-more-than-ever.md]]

**Summary:** After 18 months teaching developers to build with AI agents, Matt Pocock delivers a clear thesis: software fundamentals matter more than ever, not less. The video critiques the "specs to code" movement as "vibe coding by another name," showing how blind regeneration produces software entropy — each iteration degrades code quality. Pocock maps five classic software engineering principles to AI-assisted workflows: (1) **Design concept** (Brooks) → the "Grill Me" skill for adversarial alignment before coding; (2) **Ubiquitous language** (Evans) → shared domain terminology reducing AI verbosity; (3) **Tight feedback loops** (Beck) → TDD forcing small, verifiable steps; (4) **Deep modules** (Ousterhout) → simple interfaces hiding complex functionality so agents test at boundaries; and (5) **Entropy resistance** → daily architectural investment preventing codebase decay. The core warning: bad code is the most expensive it has ever been, because a messy codebase blocks capture of AI's benefits.

**Key contributions:**
- "Grill Me" skill: adversarial interviewing (40–100 questions) to establish shared design concept
- Ubiquitous language file (DDD-style terminology table) scanned by AI during planning and implementation
- TDD as the primary feedback loop forcing agents into small, deliberate steps
- Deep modules as the antidote to AI-generated shallow module proliferation
- Software entropy as the central risk of unguided AI regeneration
