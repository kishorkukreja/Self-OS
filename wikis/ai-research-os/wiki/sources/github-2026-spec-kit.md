# GitHub Spec Kit — Spec-Driven Development Toolkit

**Type:** repo  
**Date:** 2026-05-10  
**URL:** https://github.com/github/spec-kit  
**Raw file:** [[../raw/repos/spec-kit-2026-05-10.md]]

**Summary:** GitHub Spec Kit is an open-source toolkit for spec-driven development with coding agents. Its workflow turns project principles, specifications, plans, and tasks into first-class artifacts that an agent can implement against, rather than treating planning as disposable chat. The toolkit aligns with Self-OS/taskOS patterns such as PRDs, issue decomposition, acceptance criteria, and QA-gated implementation. It also adds a governance warning: official installation paths matter because similarly named packages can create supply-chain risk. For the wiki, the durable idea is that agent implementation quality improves when specs and principles are explicit, versioned, and available as execution inputs.

**Key contributions:**
- Provides the specify CLI plus slash-command style lifecycle for spec, plan, tasks, and implementation.
- Treats governance and project constitution as inputs before agent coding begins.
- Maps strongly to taskOS patterns for PRD-to-tasks-to-implementation workflows.
