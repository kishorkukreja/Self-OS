## Summary

The Code's 13 May issue frames programming through two linked pressures: agents are becoming more autonomous, and the ecosystem around them is becoming more exposed. Anthropic's Claude Code `/goal` command is presented as a concrete step toward evaluator-confirmed work loops, where the user specifies a target and the agent continues until a separate judge says the target is met. Prime Intellect's renderers point at the adjacent infrastructure problem: translating between token training data and message-based agent environments efficiently enough to make agent data pipelines practical.

The issue's security coverage is a reminder that agentic development does not sit outside the software supply chain. Google's report on an AI-built zero-day, malicious npm packages, and dependency-lockdown prompts all point toward a world where code-writing agents must be paired with dependency hygiene, evals, and package-review workflows. The issue also situates persistent agents such as Hermes Agent and OpenClaw as a distinct species from session-bound coding tools, with memory and long-lived execution as the architectural moat.

## Key contributions
- Documents Claude Code's move toward goal-directed autonomous coding loops.
- Links AI-assisted exploitation and malicious packages to the operational safety requirements for coding agents.
- Captures the emergence of persistent personal agents as developer infrastructure rather than one-off chat sessions.
