**Summary:** This video capture describes the practical use of long-running goal loops for coding agents. The central lesson is that simply allowing an agent to run longer is not enough; the system needs a standing goal, durable artifacts, a definition of done, and a judgment step that decides whether to continue. That maps closely to Self-OS patterns such as mission files, goal-buddy review, and reward/evaluation loops. The source is especially useful because it names the failure mode of premature success claims on broad tasks and suggests a remediation: keep objective, evidence, and continuation prompt outside the transient chat turn so the next loop can resume from disk-backed state.

**Key contributions:**
- Frames /goal-style commands as judged continuation loops, not blind retries.
- Connects goal persistence to mission documents, durable artifacts, and completion criteria.
- Provides a practical vocabulary for reducing premature agent stop conditions.
