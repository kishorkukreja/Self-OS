## Summary

The 2026-05-06 issue of The Code ties daily programming news to the operational demands of agentic coding. The headline items include GPT-5.5 Instant becoming the new ChatGPT default, Subquadratic’s SubQ long-context model with claimed sub-quadratic scaling, and Gemini API File Search adding multimodal text-and-image retrieval with inline citations. The issue’s deeper insight is about SKILL.md files: treating a skill as one large prompt can burn a large fraction of context before an agent starts, whereas a lightweight “spine” that points to reference files aligns better with staged runtime loading. That observation is directly relevant to Hermes skill maintenance because it frames skills as small programs that need evals after model upgrades.

The social and tooling sections reinforce a shift from single-agent demos to orchestrated engineering practice. Items on multi-agent orchestration, Claude Code/Codex on local hardware, and running hundreds of agents from a phone all point toward more distributed coding workflows. The issue also includes a simple pre-commit hook that asks Claude to inspect staged diffs for console logs, hardcoded secrets, and unfinished TODOs before commit. The durable takeaway is not the exact hook, but the pattern: place AI review at workflow chokepoints where humans already make irreversible changes. Combined with the OpenAI memory-and-compaction paper highlighted as the trending paper, the issue is a useful snapshot of coding agents becoming ordinary engineering infrastructure.

## Key takeaways

- Skill files should be structured as lean entrypoints with references rather than monolithic prompts.
- AI review belongs at existing control points such as pre-commit, CI, and PR review.
- The issue connects long-context models, multimodal retrieval, and multi-agent orchestration as converging agent-coding infrastructure.
