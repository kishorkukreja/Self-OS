## Summary

The Mnilax thread is a practical prompt-operations source about turning `CLAUDE.md` into a compact operating contract for coding agents. The underlying pattern is that agent performance improves when rules are written as durable, repository-local constraints covering commands, architecture, coding style, testing, security, and workflow norms. Rather than relying on the model to infer project conventions every turn, the template makes expectations explicit and reusable across sessions. For AI Research OS and Self-OS, the source belongs beside other agent-infrastructure material because it highlights the same theme as skills, state files, and receipt logs: reliable agents need structured context that is close to the code and maintained as the project changes. The template should be treated as a design reference, not blindly copied into every repository. The source is useful primarily as a compiled signal rather than as a final answer: the raw capture remains the canonical place for exact wording, examples, figures, and links. In the wiki layer it should be read as evidence about recurring operating patterns, decision pressure, and implementation considerations that can be connected to adjacent source summaries without copying the raw text verbatim.

- Connects repository-local instructions to safer, more consistent agent execution.
