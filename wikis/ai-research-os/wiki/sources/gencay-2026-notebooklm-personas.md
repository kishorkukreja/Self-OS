# I Built 4 NotebookLM Personas With 24 Prompts

**Type:** newsletter  
**Date:** 2026-04-26  
**Author:** Gencay  
**Raw file:** [[../raw/newsletters/2026-04-26-gencay-notebooklm-4-personas-24-prompts.md]]

**Summary:**  
Gencay argues that most users underutilise Google NotebookLM by treating it as a simple summariser rather than a **persona engine**. The core insight is that the same set of sources can produce radically different outputs when routed through distinct role-based prompts. The article presents four personas — Marketer, Student, Researcher, and Designer — each with a setup prompt and six working prompts tied to specific NotebookLM Studio buttons (Data Table, Reports, Audio Overview, Infographic, Slide Deck, Flash Cards, Quiz, Video Overview). A key operational rule is to keep one notebook per persona to avoid cross-domain source scrambling.

**Key contributions:**
- **Persona-engine framing** — Reframing NotebookLM from summariser to multi-role reasoning engine.
- **Four complete personas** — Marketer (competitive intel, positioning, copy audit, ICP profiling, campaign angles, audio briefs); Student (concept breakdown, Feynman explanations, active-recall flashcards, gap finding, exam prep, connection maps); Researcher (literature synthesis, gap analysis, methodology critique, citation webs, contradiction finding, hypothesis generation); Designer (infographic briefs, visual hierarchy audits, brand-voice extraction, comparison infographics, design-system audits).
- **Button-output mapping** — Each prompt is explicitly paired with a Studio output type, showing which button to press for which task.
- **Scalable skeleton** — The three-part structure (sources → setup prompt → working prompts) can be reused for any new voice (copywriter, lawyer, startup advisor, etc.).
