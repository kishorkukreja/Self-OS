# HALO — Hierarchical Agent Loop Optimization

**Type:** repo  
**Date:** 2026-05-04  
**URL:** https://github.com/context-labs/HALO  
**Raw file:** [[../raw/repos/halo-2026-05-04.md]]

## Summary
HALO is a trace-driven methodology for improving agent harnesses. It instruments executions with OpenTelemetry-compatible traces, analyzes recurring failure modes through a Recursive Language Model engine, and then feeds the resulting report to coding agents that can edit the harness. The important shift is operational: improvement is not a one-off prompt tweak but a loop of execution, observability, diagnosis, implementation, and retest. For Self-OS, HALO is a strong reference for future agent maintenance because it connects logs/traces to concrete harness edits and makes observability part of the improvement substrate. The project also provides vocabulary for separating symptoms visible in traces from the specific prompt, tool, or control-flow changes that should be tested next.

## Key contributions
- Uses execution traces to diagnose harness failure modes.
- Separates model training from harness optimization around prompts, tools, and control flow.
- Supports an operational loop where coding agents apply trace-derived improvements.
