TabPFN is a foundation model for tabular data that provides fast classification and regression for tabular datasets through a PyTorch/CUDA implementation. The current default model is TabPFN 2.6, trained on synthetic data, with older/default variants such as TabPFN 2.5 and TabPFN 2 also available. It is positioned as a high performance option for small to medium tabular problems, with best fit guidance around datasets below roughly 100,000 rows and 2,000 features, plus extensions for larger datasets and advanced workflows. The linked section is especially useful because it maps the TabPFN ecosystem : local core inference, cloud client, extensions, no code UX, and workflow decisions around GPU availability, classification/regression vs unsupervised tasks, performance tuning, interpretability, and large data limits. For Self OS and applied AI workflows, this is a candidate tool to remember for quick tabular modeling, baseline generation, business/operations datasets, anomaly detection, and explainable tabular experiments. TabPFN supports both classification and regression with a simple sklearn style API: TabPFNClassifier and TabPFNRegressor . GPU is strongly recommended; CPU is only feasible for small datasets around ≲1,000 samples. The hosted TabPFN Client is recommended when no local GPU is available and has native text support. Avoid typical preprocessing habits: do not scale features and do not one hot encode categorical variables before feeding them to TabPFN.
