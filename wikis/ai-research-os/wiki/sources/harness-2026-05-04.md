# Harness — Team-Architecture Factory for Claude Code

**Type:** repo  
**Date:** 2026-05-04  
**URL:** https://github.com/revfactory/harness  
**Raw file:** [[../raw/repos/harness-2026-05-04.md]]

## Summary
Harness is a Claude Code meta-skill for generating project-specific teams of specialized agents, skills, and coordination protocols. It is important because it treats team architecture as an artifact: pipeline, fan-out/fan-in, expert pool, producer-reviewer, supervisor, and hierarchical delegation patterns can be generated, validated, and evolved. For Self-OS, the source is a reference for turning domain descriptions into reusable agent organizations rather than hand-writing every subagent prompt. It also reinforces that multi-agent systems need explicit coordination designs, dry runs, and evolution loops to avoid becoming chaotic token sinks. The source is especially relevant when a task needs several specialists but still requires a clear protocol for handoffs, review, and consolidation.

## Key contributions
- Generates .claude agents and skills from a domain-level prompt.
- Names reusable team patterns such as pipeline, fan-out/fan-in, and producer-reviewer.
- Connects multi-agent architecture to validation and harness evolution.
