## Summary

The Code’s May 15 issue captures a turning point in developer-agent ergonomics. OpenAI’s Codex mobile preview lets developers review diffs and approve commands from a phone while computation remains on their laptop, and xAI’s Grok Build positions a terminal coding agent around plan mode, isolated worktrees, and parallel subagents. Together, these releases make AI coding more ambient and distributed: the operator no longer has to sit at the original machine to keep a run moving.

The issue balances that momentum with warnings about the agentic coding trap. Lars Faye’s critique and Anthropic’s debugging-skill data both argue that orchestration is not free leverage: developers still need enough implementation skill and system understanding to supervise generated code. The practical tooling notes reinforce that point. Prompt-cache warmup can materially reduce first-response latency, but faster agents still need scoped permissions, reviewable diffs, and traceable decisions. For Self-OS, this source supports a workflow where agents do background work and humans approve from lightweight surfaces, while skills, plans, and tests preserve enough context for meaningful supervision.

## Key contributions
- Codex mobile and Grok Build expand coding-agent control surfaces beyond a single terminal session.
- Agentic coding supervision is now a first-order productivity and skill-maintenance problem.
- Prompt caching and plan modes improve throughput only when paired with review discipline.
